Contexto
Problema e Solução
Oficinas mecânicas com processos manuais → sistema integrado com DDD
O Problema
  • Oficinas operam com planilhas e processos manuais dispersos
  • Sem rastreabilidade no ciclo de vida das ordens de serviço
  • Aprovação de orçamentos por telefone — sem registro ou segurança
  • Controle de estoque manual, sujeito a erros e divergências
  • Dificuldade para gerar métricas e relatórios operacionais
  • Acesso indiscriminado às informações de clientes e OS
A Solução (Fase 1)
  • API REST completa — 50+ endpoints documentados com Swagger
  • Máquina de estados para OS: 7 estados + CANCELADA
  • Aprovação de orçamento via link seguro (HMAC-SHA256)
  • Estoque com reserva automática ao aprovar o orçamento
  • JWT + autorização híbrida (role + object-level security)
  • Domain Events para notificações e efeitos colaterais

Regra de Dependência
  • Setas sempre apontam para o domínio — nunca o contrário
  • Ports: interfaces Java em domain/repository/
  • Adapters JPA em infra/adapter/Jpa*RepositoryAdapter
  • Dois mappers: infra/mapper/ e presentation/mapper/
  • Domain layer verificada: zero import Spring/JPA/HTTP

Modelo de Domínio
OrdemServico & Domain Events
Aggregate root central · Máquina de estados · 7 eventos de domínio
Máquina de Estados — OrdemServico (Aggregate Root)
RECEBIDA → EM_DIAGNOSTICO → AGUARDANDO_APROVACAO → APROVADA
RECEBIDA → EM_DIAGNOSTICO  → EM_EXECUCAO → FINALIZADA → ENTREGUE
CANCELADA ← transição válida a partir de qualquer estado

Transições inválidas lançam TransicaoStatusInvalidaException (HTTP 422). Ações via sub-recurso: POST /api/ordens-servico/{id}/acoes (ADR-003).

Domain Events (Spring ApplicationEvent)
Evento Handler Efeito
OsCriadaEvent OsCreatedListener Log / auditoria da criação
OrcamentoGeradoEvent OrcamentoEmailListener Envia email ao cliente com link HMAC-SHA256
OrcamentoAprovadoEvent EstoqueListener Baixa estoque das peças reservadas (ADR-019)
OrcamentoReprovadoEvent OsListener Cancela reservas de estoque
OsFinalizadaEvent PdfListener Gera PDF de entrega automaticamente
OrdemServicoCanceladaEvent EstoqueListener Estorna itens ao estoque
OrdemServicoAguardandoAprovacaoEvent OsAguardandoListener Notificação / log de estado

Todos os handlers usam @TransactionalEventListener + @Async — efeitos colaterais fora da transação principal.

ADR-022 — Fluxo Invertido: A OS avança para APROVADA quando o cliente aprova o orçamento (e não o contrário). ADR-019: o estoque é baixado nesse momento — nunca antes, nunca depois.
